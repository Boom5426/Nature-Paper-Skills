# Repository And Identifier Routing

Use this file when deciding where supporting data should live and what identifier type should anchor the statement.

## Preferred Order

Prefer this order:

1. discipline-specific public repository
2. controlled-access domain repository
3. generalist repository with DOI
4. institutional repository with stable record
5. supplement-only placement for genuinely small support files

Avoid personal websites, temporary cloud links, or email-only access routes.

## Identifier Rules

Prefer:
- accession numbers for domain repositories
- DOI or other durable persistent identifiers for deposited records
- exact supplementary file names when the journal package uses source-data files

Do not invent:
- DOI
- accession number
- reviewer link
- embargo date
- repository record URL

## Access-Route Checks

For each dataset, ask:

- Which result depends on this dataset?
- Can a reader tell where to get it?
- Is the route durable beyond one author's email address?
- If the data are restricted, who evaluates access and under what rule?
- Is reused public data cited precisely enough to reproduce the fetch?
